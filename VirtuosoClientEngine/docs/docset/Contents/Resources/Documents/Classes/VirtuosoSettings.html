VirtuosoSettings Class Reference

Inherits from NSObject
Declared in VirtuosoSettings.h

Overview

A singleton object that manages all SDK configuration settings. Receives default settings and business rules from the Backplane.

Warning: You must never instantiate an object of this type directly. The VirtuosoSettings is a singleton and should only be accessed via the provided instance method.

Utility

+ instance

The singleton instance access method

+ (VirtuosoSettings *)instance

Return Value

Returns the VirtuosoSettings object instance.

Discussion

You must never instantiate a local copy of the VirtuosoSettings object. This object is intended to be a static singleton accessed through the instance method only. Instantiating a local copy will throw an exception.

Backplane-Provided Configuration

  defaultOfflineViewingPeriod

Once the time since last Backplane sync exceeds this value, Virtuoso makes the downloaded assets inaccessible

@property (nonatomic, readonly) long long defaultOfflineViewingPeriod

Discussion

Virtuoso enforces that clients must sync with the Backplane at a required interval, or else lose access to downloaded assets. Virtuoso marks the time since the sync with the Backplane, and makes all downloaded assets inaccessible if the time exceeds this value. After this timeout, any attempts to playback any assets via the file proxy will fail. Any calls to VirtuosoAsset methods that would retrieve file paths for downloaded assets will return nil. Note that Virtuoso doesnâ€™t delete all downloaded assets; it just makes the assets inaccessible. Once the next Backplane sync occurs, the assets are again accessible.

  defaultExpiryAfterDownload

Amount of time, in seconds, between when Virtuoso finishes downloading an asset and when it deletes that downloaded asset.

@property (nonatomic, readonly) long long defaultExpiryAfterDownload

Discussion

As soon as Virtuoso finishes downloading an asset, it marks the asset with a timestamp. As soon as possible after ‘defaultExpiryAfterDownload’ has elapsed, Virtuoso will automatically delete the downloaded asset. Values less than or equal to zero mean no expiry.

  headroom

Returns the headroom configuration value.

@property (nonatomic, readonly) long long headroom

Return Value

The headroom setting (in MB).

Discussion

Headroom is the amount of free space that Virtuoso must preserve on disk. For instance, if headroom is 1GB and the device has 1.2GB free disk space, Virtuoso will begin to download a 150MB asset, but it will NOT begin to download a 500MB asset. The default is 1 GB.

  maxDownloadedAssets

The maximum number of downloads that can exist on the device at any given time.

@property (nonatomic, readonly) long long maxDownloadedAssets

Discussion

Virtuoso limits the total number of downloads the device can have on disk at any given time. Any download requests beyond this limit are queued, but will not download, and the engine will report an error indicating the download queue is blocked until the user manually deletes enough of the currently downloaded assets to get below the limit again. The default value is 100.

  useStreamPackagerForBackgroundDownloads

Enables Penthera’s StreamPackager service for background download of segmented VirtuosoAssets (HLS, HSS, DASH, etc).

@property (nonatomic, assign) Boolean useStreamPackagerForBackgroundDownloads

Discussion

Penthera’s StreamPackager (a server proxy) allows Virtuoso to more efficiently download segmented VirtuosoAsset formats even when the enclosing app isn’t in the foreground. To disable this feature (and continue with slower background downloads), set this property to NO. Default is YES.

Warning: Since Virtuoso uses this value during startup, you must supply this value before calling any other Virtuoso methods. Changes to this value won’t take effect until the next time you call startup.

  minimumBackplaneSyncInterval

The minimum amount of time between Backplane syncs

@property (nonatomic, assign) NSTimeInterval minimumBackplaneSyncInterval

Discussion

No matter how many times the Backplane sync method is called, or how often internal events might attempt a sync, the SDK will generally sync with the backplane more often than this value. Certain time-critical events, such as a SDK push notice, may cause the SDK to sync regardless of the last sync. The default is 15 minutes.
